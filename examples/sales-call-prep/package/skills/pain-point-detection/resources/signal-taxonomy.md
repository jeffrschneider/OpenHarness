# Pain Point Signal Taxonomy

## Signal Categories by Source

### Job Postings

| Pattern | Pain Point | Your Angle |
|---------|------------|------------|
| Multiple similar roles | Team overwhelmed | Force multiplier |
| "Process improvement" | Manual processes | Automation |
| "Data" + "operations" | Data chaos | Integration |
| Urgent language | Acute problem | Speed to value |
| Entry-level hiring spree | Scaling basics | Efficiency tools |
| Senior hire | Strategic gap | Partnership |

|---------|---------------|-------|
| "Tool sprawl" | Too many disconnected tools | Consolidation |

| "Scaling pains" | Growth outpacing systems | Scale-ready solutions |
| "Legacy systems" | Technical debt | Modernization |

## Pain Point Categories

### Operational Pain
- Manual processes eating time
- Tool sprawl/fragmentation
- Data silos
- Compliance complexity
- Scaling bottlenecks

### Financial Pain
- Cost overruns
- Budget pressure
- ROI not realized from current tools
- Hidden costs (maintenance, integration)

### Strategic Pain
- Competitive pressure
- Market changes
- Growth constraints
- Innovation lag

### People Pain
- Team burnout
- Skill gaps
- Hiring challenges
- Retention issues

## Signal-to-Message Mapping

| Signal | Pain Point | Message Theme |
|--------|------------|---------------|

| Series C announcement | Growth pressure | "Built for scale" |
