# Job Posting Signal Analysis

## How to Read Job Postings for Sales Intelligence

### Quantity Signals

| Number of Similar Posts | Interpretation |
|------------------------|----------------|
| 1 | Replacement hire, normal turnover |
| 2-3 | Team growth, initiative starting |
| 4+ | Major scaling, urgent need |
| 10+ | Transformation, massive investment |

### Seniority Signals

| Level Hiring | Interpretation |
|--------------|----------------|
| Entry-level | Building capacity |
| Mid-level | Execution need |
| Senior/Lead | Expertise gap |
| Director+ | Strategic priority |
| VP/C-level | Organizational change |

### Urgency Language

**High urgency indicators:**
- "Immediate start"
- "ASAP"
- "Contract-to-hire"
- "Competitive salary"
- "Signing bonus"

**Normal indicators:**
- Standard posting language
- No timeline mentioned

### Keyword Analysis by Domain

#### Operations/Process
- "Process improvement" = Manual work problems
- "Automation" = Repetitive task burden
- "Efficiency" = Cost pressure
- "Scale" = Growth outpacing systems

#### Data/Analytics
- "Data operations" = Data chaos
- "Data quality" = Trust issues
- "Data integration" = Silos
- "Analytics engineer" = Insight hunger

#### Technology
- "Modernization" = Legacy systems
- "Cloud migration" = Infrastructure change
- "Security" = Risk awareness
- "Integration" = Tool sprawl

### Tech Stack Mentions

**Tools mentioned in job posts reveal:**
- What they currently use (evaluate fit)
- What they're adopting (timing of change)
- Skill gaps (why they're hiring)

**Example:**
> "Experience with Salesforce required, HubSpot a plus"
> = They use Salesforce, may be considering HubSpot

### Job Posting Sources

| Source | Freshness | Coverage |
|--------|-----------|----------|
| Company careers page | Most current | Direct jobs only |
| LinkedIn | Fresh | Professional roles |
| Indeed/Glassdoor | Variable | Broader roles |
| AngelList/Wellfound | Fresh | Startup focus |
| Industry job boards | Variable | Specialized roles |

### Analysis Template

For each relevant posting, capture:

```
Role: [Title]
Posted: [Date]
Key Requirements: [Skills/tools mentioned]
Pain Signal: [What problem does this role solve?]
Your Angle: [How does your product relate?]
Urgency: [High/Medium/Low]
```

### Red Flags

- Same role posted repeatedly over months = retention problem
- Very long requirements list = unrealistic expectations
- Below-market salary mentioned = budget constraints
- "Wearing many hats" = resource constrained
